1. 배열

1-1. 배열이란 무언인가

자료형이 같은 변수들을 모아놓은 집합입니다. 인덱스는 언어에 따라 다르지만, 대부분 0을 시작으로 두고 있습니다. 선언에 따라 1차원부터 고차원까지 다양한 배열 선언이 가능합니다.

1-2. 베열의 예시

여기선 1차원 배열과 이차원 배열을 다룹니다.

  • 1차원 배열(각 상자들을 클릭해보세요.)

    니지동 멤버 배열[]=

    이렇게 []안에 들어가는 숫자를 배열의 인덱스라 합니다. 아까 전에도 언급했다 싶이, 배열의 시작은 0이기 때문에 전체 갯수 n-1개까지 나타냅니다.

  • 2차원 배열(각 상자들을 클릭해보세요.)

    0번째 줄
    2번째 줄
    3번째 줄

    니지동 멤버 배열[][]=

    다차원 배열은 인간이 사용하기 편하게, 가상의 차원을 만들어 행과 열을 이용해 표현하는 프로그래밍 기법입니다. 프로그래머의 표현 방법에 따라 (y,x)나 (x,y)의 설정이 자유롭지만, 대부분 후자를 따르고 있습니다.

  • 2.메모리에 저장되는 방식

    배열의 이름과, 첫 인덱스의 주소값은 같습니다. 배열이 선언된 자료형마다 사이즈가 하나씩 더해져, 그 메모리 주소에 저장됩니다. 여기선 하나당 4바이트라고 가정하겠습니다. 2차원 배열 역시 저장될때 이런 형태로 나열되어 저정됩니다.

    선택한 배열의 주소(배열 이름의 주소:FF0):

    또한 배열은 파라미터로 넘길때, Call by Refence 즉 주소값이 복사됩니다. 코드를 보면, 메인함수에서 넘긴 배열의 주소와, 파라미터로 받은 배열의 주소가 같음을 알수 있습니다. 배열을 파라미터로 넘길때, 받는 함수는 그 배열의 크기를 알 수 없으니, 배열의 크기도 같이 넘겨줘야합니다.

2. 실험

2-1. 예제코드

예제코드 보기

실험대상:n을 n번 더하는 계산. (단, for루프 제어연산과 대입연산은 고려하지 않고, 컴퓨터 성능으로 인해 어지간한 계산은 차이가 잘 나지 않으니, 한번 연산 당 변수의 값을 올림(변수 second))

  • 1.n*n

    void firstcase(int randomNumber) {
    int second=0;
    int sum=randomNumber*randomNumber;
    second++;
    std::cout<<(sum);
    std::cout<<(second);
    }

  • 2.n을 반복문을 통해 n번 더함

    void secondcase(int randomNumber) {
    int sum=0;
    int second=0;
    for(int i=0;i<randomnumber;i++){
    sum+=randomNumber;
    second++;
    }
    std::cout<<(sum);
    std::cout<<(second);
    }

  • 3.n을 이중반복문을 통해 1씩 증가시킴

    void thirdCase(int randomNumber) {
    int sum=randomNumber*randomNumber;
    int second=0;
    for(int i=0;i<randomnumber;i++){
    for(int j=0;j<randomnumber;j++){
    sum++;
    second++;
    }
    }
    std::cout<<(sum);
    std::cout<<(second);
    }

2-2. 각 코드별 알고리즘 복잡도 및 예상 그래프

그래프 보기

  • 첫번째 경우

    값의 상관 없이 한번만 연산된다. O(1)

  • 두번째 경우

    값의 크기만큼 반복문이 돌며 연산된다. O(n)

  • 세번째 경우

    값의 제곱 크기만큼 반복문이 돌며 연산된다. O(n^2)

3. 느낀점

소스코드를 단순히 구현했을때는, 어떤 알고리즘이 더욱 빠르고 자원을 덜 잡아먹는지 고민하지 않았다. 하지만, 이번 수업을 통해 어떤 방식을 채택하느냐에 따라, 천차만별의 속도가 나오는 것을 확인하여, 나중에 무언가를 프로그래밍을 통해 구현하고자 할때, 이런 걸 알아두는 것이 더 좋은 프로그래머가 되기 위함 자질임을 알 수 있어 좋았다.
